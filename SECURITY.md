## Reporting a Vulnerability

We take the security of TransmitFlow seriously. If you have found a security vulnerability in this project, please report it to us.

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to `shubhampardule@gmail.com`. You should receive a response within 48 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

Please include the following details in your report:

- Type of issue (e.g. cross-site scripting, SQL injection, remote code execution)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

### Prefer Secure Communication

If possible, please encrypt your output with PGP. Key ID: `[Your Key ID Here]` (if applicable).

### Our Response Policy

We will:

1. Acknowledge your report within 48 hours.
2. Investigate the issue to confirm the vulnerability.
3. If confirmed, we will work on a patch.
4. We will release a security update as soon as possible.
5. We will credit you for the discovery (unless you prefer to remain anonymous).

Thank you for helping keep TransmitFlow and its users safe!
